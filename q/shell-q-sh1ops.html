Shell 脚本在后端日常运维中怎么用？写过哪些实用脚本？

中等 scene

后端工程师常用的 Shell 场景

  • 日志分析：grep + awk + sort + uniq -c 快速统计错误分布、慢请求 Top N
  • 批量操作：批量重启服务、批量更新配置、批量清理过期文件
  • 健康检查：定时检测端口、进程、磁盘空间，异常时发告警
  • 部署脚本：编译、打包、推送、滚动重启的自动化串联

实用示例

# 统计最近 1 小时 5xx 错误的 Top 10 接口
grep '$(date +%H):' access.log | awk '$9 >= 500 {print $7}' | sort | uniq -c | sort -rn | head -10

# 批量检查多台机器的服务状态
for host in $(cat hosts.txt); do
  ssh $host 'systemctl is-active myservice' 2>/dev/null || echo "$host: DOWN"
done

# 清理 7 天前的日志
find /var/log/app/ -name '*.log' -mtime +7 -exec rm -f {} \;

写 Shell 脚本的工程习惯

  • 开头加 set -euo pipefail，遇错即停，未定义变量报错
  • 关键操作前做 dry-run 或加确认提示
  • 日志输出带时间戳，方便事后排查
面试时别只说\"会用 grep\"，能举一个你实际写过的脚本场景（比如日志分析、批量部署），说清输入输出和用到的命令组合。
