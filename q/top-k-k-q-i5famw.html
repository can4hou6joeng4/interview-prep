实现 Top K 问题（海量数据中找前 K 大的元素）

中等

思路

维护一个大小为 k 的小顶堆。遍历所有元素时，如果当前值大于堆顶，就替换堆顶并重新下沉。遍历结束后，堆里保留的就是前 K 大元素。

type MinHeap []int

func topK(nums []int, k int) []int {
    h := &MinHeap{}
    heap.Init(h)
    for _, num := range nums {
        if h.Len() < k {
            heap.Push(h, num)
            continue
        }
        if num > (*h)[0] {
            heap.Pop(h)
            heap.Push(h, num)
        }
    }
    return *h
}
时间复杂度 O(n log k)，空间复杂度 O(k)，这是面试里最标准的 Top K 解法。
