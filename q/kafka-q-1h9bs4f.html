Kafka 如何保证消息顺序性？全局有序和分区有序的区别？

中等

Kafka 的顺序保证

  • 分区内有序：单个 Partition 内的消息严格有序（追加写入）
  • 跨分区无序：不同 Partition 之间没有顺序保证

需要顺序的场景怎么做

// 方案一：同一个 Key 的消息发到同一个 Partition
producer.Produce(&kafka.Message{
    TopicPartition: kafka.TopicPartition{Topic: &topic},
    Key:   []byte(orderId),   // 用 orderId 做 Key
    Value: payload,
})
// Kafka 对 Key 做 hash % partitionCount，相同 Key 固定到同一 Partition

// 方案二：全局有序（极端场景）
// Topic 只设 1 个 Partition（牺牲并行度，不推荐）

顺序消费注意

  • Consumer 内不能用多线程并行处理同一个 Partition 的消息（否则乱序）
  • 如需并行：按 Key 分组，相同 Key 的消息由同一个 goroutine 处理
岗位关联：腾讯赛事平台、百度广告平台都重度依赖 Kafka 做事件驱动，订单/支付/广告投放事件必须保证分区有序
