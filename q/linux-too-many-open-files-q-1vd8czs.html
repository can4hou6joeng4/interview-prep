Linux 文件描述符是什么？Too many open files 怎么解决？

简单 scene

文件描述符 (fd)

Linux 中一切皆文件。每个进程打开的文件、socket、管道等都有一个整数编号（fd）

  • 0：stdin | 1：stdout | 2：stderr
  • 每个进程有 fd 上限（默认 1024）

Too many open files 排查

# 查看当前进程打开的 fd 数
ls /proc/PID/fd | wc -l

# 查看进程的 fd 限制
cat /proc/PID/limits | grep "open files"

# 查看系统级限制
cat /proc/sys/fs/file-max

# 解决方案
# 1. 临时调整
ulimit -n 65535

# 2. 永久调整 (/etc/security/limits.conf)
* soft nofile 65535
* hard nofile 65535

# 3. systemd 服务
[Service]
LimitNOFILE=65535

根本原因

通常不是真的需要这么多 fd，而是代码有资源泄漏：

  • HTTP 响应 Body 未 resp.Body.Close()
  • 数据库连接未释放
  • 文件打开后未关闭
Go 中养成习惯：defer resp.Body.Close()、defer rows.Close()、defer f.Close()
