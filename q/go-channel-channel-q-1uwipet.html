Go 中 Channel 的底层结构是什么？有缓冲和无缓冲 Channel 的区别？

简单

底层结构 (runtime.hchan)

type hchan struct {
    qcount   uint      // 当前队列中的元素数量
    dataqsiz uint      // 环形缓冲区大小（即 cap）
    buf      unsafe.Pointer // 环形缓冲区指针
    elemsize uint16    // 元素大小
    closed   uint32    // 是否关闭
    sendx    uint      // 发送索引
    recvx    uint      // 接收索引
    recvq    waitq     // 等待接收的 goroutine 队列
    sendq    waitq     // 等待发送的 goroutine 队列
    lock     mutex     // 互斥锁
}

无缓冲 vs 有缓冲

  • 无缓冲 (make(chan T))：发送和接收必须同时就绪，否则阻塞。本质是同步通信
  • 有缓冲 (make(chan T, n))：缓冲区未满时发送不阻塞，缓冲区为空时接收阻塞。本质是异步通信

关键行为

  • 向 nil channel 发送/接收会永久阻塞
  • 向已关闭的 channel 发送会 panic
  • 从已关闭的 channel 接收会返回零值（可通过 v, ok := <-ch 判断）
