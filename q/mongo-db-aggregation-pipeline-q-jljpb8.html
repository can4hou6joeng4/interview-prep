MongoDB 的聚合管道 (Aggregation Pipeline) 是什么？常用的阶段有哪些？

中等

聚合管道

类似 Unix 管道，文档依次通过多个阶段 (Stage)，每个阶段对文档做一种变换

db.orders.aggregate([
    { $match: { status: "paid" } },              // WHERE
    { $group: {                                    // GROUP BY
        _id: "$storeId",
        totalAmount: { $sum: "$amount" },
        count: { $sum: 1 }
    }},
    { $sort: { totalAmount: -1 } },               // ORDER BY
    { $limit: 10 },                                // LIMIT
    { $lookup: {                                   // LEFT JOIN
        from: "stores",
        localField: "_id",
        foreignField: "_id",
        as: "store"
    }},
    { $project: {                                  // SELECT
        storeName: { $arrayElemAt: ["$store.name", 0] },
        totalAmount: 1,
        count: 1
    }}
])

常用阶段

  • $match：过滤（尽量放前面，利用索引）
  • $group：分组聚合（sum/avg/count/max/min）
  • $sort / $limit / $skip：排序分页
  • $lookup：类似 LEFT JOIN（跨集合关联）
  • $unwind：展开数组（一条文档变多条）
  • $project / $addFields：字段选择/计算
