常用的 Linux 排查命令？线上服务 CPU 飙高怎么定位？

简单 scene

常用命令

  • top / htop：实时查看 CPU、内存占用，按进程排序
  • ps aux | grep xxx：查看进程信息
  • netstat -tlnp / ss -tlnp：查看端口监听和连接状态
  • lsof -i :8080：查看端口被哪个进程占用
  • df -h / du -sh：磁盘空间
  • free -h：内存使用
  • tail -f /var/log/xxx.log：实时查看日志
  • strace -p PID：跟踪系统调用

CPU 飙高排查（Go 服务）

# 1. top 找到 CPU 最高的进程 PID
top -c

# 2. 对 Go 服务，用 pprof
# 在代码中引入
import _ "net/http/pprof"
go http.ListenAndServe(":6060", nil)

# 3. 采集 CPU profile
go tool pprof http://localhost:6060/debug/pprof/profile?seconds=30

# 4. 查看火焰图
go tool pprof -http=:8081 profile.out

# 5. 常见原因：
# - 死循环 / 无限递归
# - GC 压力大（大量小对象分配）
# - 锁竞争（mutex contention）
# - 正则表达式回溯
