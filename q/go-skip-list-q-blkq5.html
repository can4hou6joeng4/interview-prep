实现一个 Go 版本的跳表（Skip List），为什么它适合有序集合？

困难

核心结构

跳表本质上是“多层有序链表”。底层保存完整数据，上层作为索引层，查询时先从高层快速跳，再逐层下探。

type Node struct {
    score float64
    value string
    next  []*Node
}

func search(head *Node, score float64) *Node {
    cur := head
    for level := len(head.next) - 1; level >= 0; level-- {
        for cur.next[level] != nil && cur.next[level].score < score {
            cur = cur.next[level]
        }
    }
    return cur.next[0]
}

为什么适合有序集合

  • 查询、插入、删除的平均复杂度都是 O(log n)
  • 比平衡树更容易实现区间遍历和顺序扫描
  • Redis 的 ZSet 在一定规模下就采用了 skiplist + hash 的组合
