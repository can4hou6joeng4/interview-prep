容器健康检查 + 优雅关机如何配合 K8s 的滚动更新？

中等 scene

完整链路

# K8s Deployment 配置
spec:
  containers:
  - name: app
    livenessProbe:     # 存活探针：失败则重启容器
      httpGet: { path: /healthz, port: 8080 }
      periodSeconds: 10
    readinessProbe:    # 就绪探针：失败则从 Service 摘除
      httpGet: { path: /readyz, port: 8080 }
      periodSeconds: 5
    lifecycle:
      preStop:          # 在 SIGTERM 前执行
        exec:
          command: ["sh", "-c", "sleep 5"]  # 等待 LB 摘掉流量
  terminationGracePeriodSeconds: 30

滚动更新流程

  1. K8s 发送 SIGTERM 给 Pod，同时将 Pod 从 Service Endpoints 移除
  2. preStop hook 执行（sleep 5s 等 LB 生效）
  3. 应用收到 SIGTERM，readiness 返回不健康，开始优雅关机
  4. 等待处理中的请求完成（你的 10 秒超时保护）
  5. 关闭 DB/Redis 连接，进程退出
  6. 超过 terminationGracePeriodSeconds 仍未退出则 SIGKILL 强杀
