服务雪崩是什么？熔断、降级、限流怎么配合？

中等 scene

服务雪崩

服务 A 调用服务 B，B 响应慢 → A 的线程/goroutine 被占满 → A 也变慢 → 调用 A 的 C 也受影响 → 雪崩

三板斧

  • 限流 (Rate Limiting)：入口处控制流量，超过阈值直接拒绝。防止过载
  • 熔断 (Circuit Breaker)：检测到下游错误率过高时，暂停调用一段时间（快速失败），避免无效重试。状态：Closed → Open → Half-Open
  • 降级 (Fallback)：熔断后返回兜底数据（缓存数据/默认值），而非直接报错
// Go 熔断器（简化版）
type CircuitBreaker struct {
    failures    int
    threshold   int       // 连续失败 N 次后熔断
    state       string    // closed / open / half-open
    lastFailure time.Time
    cooldown    time.Duration
}

func (cb *CircuitBreaker) Call(fn func() error) error {
    if cb.state == "open" {
        if time.Since(cb.lastFailure) > cb.cooldown {
            cb.state = "half-open" // 试探一次
        } else {
            return ErrCircuitOpen // 快速失败
        }
    }
    err := fn()
    if err != nil {
        cb.failures++
        cb.lastFailure = time.Now()
        if cb.failures >= cb.threshold { cb.state = "open" }
        return err
    }
    cb.failures = 0
    cb.state = "closed"
    return nil
}
