AES-CBC + RSA 混合加密的流程？为什么不直接用 RSA 加密全部数据？

中等 project

为什么不直接用 RSA

  • RSA 加密数据长度受限（2048 位密钥最多加密 245 字节）
  • RSA 加密速度极慢（比 AES 慢约 1000 倍）

混合加密流程

// 加密
1. 随机生成 AES-256 密钥 (32 bytes) 和 IV (16 bytes)
2. 用 AES-CBC 加密明文数据（对称，快速，无长度限制）
3. 用 RSA 公钥加密 AES 密钥（非对称，只加密 32 bytes）
4. 发送：RSA(AES_Key) + IV + AES_CBC(Data)

// 解密
1. 用 RSA 私钥解密出 AES 密钥
2. 用 AES 密钥 + IV 解密数据

CBC 模式注意事项

  • 需要 PKCS7 Padding（明文长度对齐到块大小 16 字节）
  • IV 必须每次随机生成，不能复用（否则相同明文产生相同密文）
  • 生产中建议用 AES-GCM（自带认证，防篡改）替代 AES-CBC
