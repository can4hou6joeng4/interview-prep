Operator 的 Reconcile 循环是什么？为什么它必须保证幂等？

困难 scene

Reconcile 的职责

Controller 收到资源变更事件后，会进入 Reconcile：读取期望状态，读取实际状态，然后不断执行修正动作，直到两者一致。

func (r *RedisReconciler) Reconcile(ctx context.Context, req ctrl.Request) (ctrl.Result, error) {
    desired := getDesiredSpec(req)
    actual := queryClusterState(req)
    diff := compare(desired, actual)
    if diff.needScale {
        scaleStatefulSet(...)
    }
    if diff.needConfigUpdate {
        updateConfigMap(...)
    }
    return ctrl.Result{}, nil
}

为什么必须幂等

  • Reconcile 可能被反复触发，甚至同一个对象短时间多次进入循环
  • 如果逻辑不幂等，就可能重复创建资源、重复发通知或把状态改乱
  • 好的 Reconcile 不依赖“上次执行到哪一步”，而是每次都基于当前真实状态重新收敛
