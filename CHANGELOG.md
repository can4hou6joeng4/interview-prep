# Changelog

本文件记录仓库中对用户可见、对仓库质量有明显影响的更新。

## 2026-04-09

### Added

- 基于 BOSS 直聘招聘方主动打招呼的 3 个岗位 JD（灏仟亿/恩士迅/中软国际），新增 6 道正式题，覆盖工作流引擎与规则引擎、WMS/TMS 物流履约、AI 数据清洗管道、Redis 内部实现机制、Shell 运维脚本、Code Review 规范。

### Improved

- CHANGELOG 补齐 2026-04-09 记录。
- index.html 最近更新面板同步本次新增。
- sitemap.xml 补充 lastmod 标签。
- 404.html 语言属性从 en 改为 zh-CN，与主站一致。
- 部分答案补充缺失的 key-point 面试提示。
- 补充缺失的 project 标签到涉及简历项目的题目。

## 2026-04-08

### Added

- 新增 JD coverage audit 脚本与关键词配置，可用脚本扫描常见岗位关键词和当前题库的覆盖强弱。
- 基于 coverage audit 再新增 3 道正式题，补齐分布式存储、Go 网络框架选型和性能优化方法论 3 个高频缺口。
- 新增专项复习入口，支持在当前学习路径内按错题、模糊题、薄弱题和已掌握题快速切换复习范围。
- 新增学习路径专题入口，将现有题库编成跨境交易、平台治理、实时系统、项目深挖和面试冲刺 5 条训练路径。
- 新增按标签过滤，支持在卡片、列表和模拟面试模式中按“简历项目 / 场景题”快速缩小题目范围。
- 继续吸收广州 shortlist backlog，再新增 4 道正式题，覆盖资源管理系统建模、配置中心分层覆盖、实时社交协议选型，以及高并发容量评估。
- 继续吸收广州 shortlist backlog，再新增 4 道正式题，覆盖金额分摊与尾差处理、订单快照设计、语音房/直播间核心模块，以及程序化广告/实时统计链路。
- 继续吸收广州 shortlist 的高信号 backlog，再新增 4 道正式题，覆盖自动化工单系统、时序数据库选型边界、压测方案设计，以及告警到回滚止损的排障闭环。
- 结合 5 篇后端 / AI Agent 面试复盘文章，再新增 5 道正式题，覆盖 goroutine vs 线程、Agent 框架选型、Agent 失败场景治理、ReAct/CoT/ToT 取舍、企业客服 Agent 分层架构与幻觉防控。
- 基于真实面试经历新增 8 道正式题，覆盖 WaitGroup、select 调度、协程池、sync.Map 引用安全、InnoDB/MyISAM 的 `count(*)` 差异、Redis 宕机降级、Redis List 与 MQ 选型、gRPC 底层机制。
- 新增 `模拟面试` 模式，支持 `掌握回顾 / 冲刺提升 / 真实混合` 三种题源与 `5 / 8 / 12` 题量。
- 模拟结果页新增 `薄弱题再练一轮`，可直接基于本轮模糊题、被打回题和未作答题继续练。
- 站点新增 `最近更新` 面板，公开展示最近几轮功能迭代。

### Improved

- `jd-questions.md` 新增 coverage audit 快照，把 recurring JD 的 strong / weak / missing 结果沉淀成下一轮扩题 backlog。
- 模拟面试 summary 新增“去薄弱专项”入口，让本轮答题结果写回后可以直接跳回薄弱题清单继续回炉。
- 切换学习路径时会自动回到一组更干净的默认筛选状态，让专题入口真正成为新的训练起点，而不是叠加旧条件后难以理解的空结果。
- 模拟结果页补充薄弱分类统计和下一轮训练建议，让结果页从报告终点变成训练指导页。
